# Synthetic 2026 Data Exclusion Guide

**Date:** April 30, 2026  
**Issue:** Synthetic 2026 movies show unrealistic vote distributions  
**Solution:** Exclude from analysis using `WHERE startYear < 2026`  
**Status:** DOCUMENTED & IMPLEMENTED

---

## PROBLEM IDENTIFIED

### Step 7 Output Analysis (Databricks v8)
When analyzing top-rated movies (Step 7), the results showed:
- **All ratings:** 9.4-9.9 (artificially high)
- **Max votes:** 3,123 (unrealistically low for top movies)
- **Expected:** Real IMDb top movies have 100K-500K+ votes

### Root Cause
The synthetic 2026 data generated by `generate_synthetic_2026_v2.py` has:
- ✗ Unrealistic rating distribution (too many 9.5+ ratings)
- ✗ Vote counts too low (avg 9,830 vs real avg 50K+)
- ✗ Not representative of actual theatrical movie patterns

### Why It Matters
- Analysis findings would be distorted by synthetic data outliers
- Investment recommendations based on this data would be unreliable
- Statistical tests would overfit to synthetic patterns

---

## SOLUTION: EXCLUDE 2026 DATA

### Filter Strategy
Add `WHERE startYear < 2026` to all analysis queries to:
- ✓ Remove synthetic 2026 movies from analysis
- ✓ Keep historical (pre-2026) data for statistical validation
- ✓ Preserve streaming merge infrastructure (Step 14)
- ✓ Ensure findings are based on realistic data patterns

### When NOT to Filter
- **Step 14:** Streaming merge — document that merge worked but don't use 2026 in analysis
- **Documentation:** Keep 2026 data comments for transparency
